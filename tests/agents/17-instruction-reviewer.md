# 테스트 #17: instruction-reviewer — 지침 리뷰어

## 대상 에이전트
`.github/agents/instruction-reviewer.agent.md`

## 사용법
에이전트 선택기 → **instruction-reviewer** 선택 후 아래 입력

## 프롬프트

```
현재 .github/ 아래의 모든 커스터마이징 파일을 점검해줘.
품질, 일관성, 중복, 누락을 모두 확인해줘.
```

## 검증 포인트

- [ ] **읽기 전용**: 파일을 수정하거나 생성하지 않고, 분석만 수행하는가
- [ ] **프론트매터 검증**: 각 파일의 `description`, `applyTo`, `name` 등 YAML 프론트매터를 점검하는가
- [ ] **일관성 분석**: 톤, 구조, 용어 사용이 파일 간 일관적인지 확인하는가
- [ ] **중복 탐지**: 동일 지침이 여러 파일에 반복되는지 감지하는가
- [ ] **누락 확인**: 참조된 파일이 실제로 존재하는지, 빈 섹션은 없는지 점검하는가
- [ ] **상호 참조 검증**: `copilot-instructions.md`에 언급된 파일이 모두 존재하는지 확인하는가
- [ ] **품질 보고서 형식**: 체계적인 보고서 형식으로 결과를 제시하는가
- [ ] **개선 제안**: 문제점에 대한 구체적인 개선 방향을 제시하는가
