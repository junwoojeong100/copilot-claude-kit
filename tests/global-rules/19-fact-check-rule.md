## 대상 규칙
`.github/copilot-instructions.md` — "팩트체크 규칙" 섹션

## 규칙 내용
> 모든 답변의 마지막에 반드시 1회 팩트체크를 수행합니다.
> 답변에 포함된 핵심 사실적 주장 3~7개를 선별하여 검증합니다.
> 검증 결과는 `### 🔍 팩트체크` 표로 답변 끝에 표시합니다.

## 프롬프트

```
Kubernetes와 Docker Swarm의 차이점을 기술적으로 비교해줘.
```

## 검증 포인트

- [ ] **팩트체크 표 존재**: 답변 마지막에 `### 🔍 팩트체크` 섹션이 반드시 포함되는가
- [ ] **3~7개 주장 선별**: 지나치게 적거나 많지 않은 수의 주장이 검증되는가
- [ ] **검증 결과 아이콘**: ✅ / 🟡 / ⚠️ / ❌ 등의 검증 결과가 표시되는가
- [ ] **표 형식 준수**: 마크다운 표 형식을 사용하는가
- [ ] **기술적 사실 검증**: 버전 번호, 기능 지원 여부, 아키텍처 설명 등을 검증하는가
