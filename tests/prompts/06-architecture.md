# 테스트 #6: architecture.prompt.md — 아키텍처 설계

## 대상 파일
`.github/prompts/architecture.prompt.md`

## 사용법
📎 → Prompt → **architecture** 선택 후 아래 입력

## 프롬프트

```
실시간 채팅 서비스를 설계해줘.
동시 접속 10만 명, 메시지 지연 100ms 이하, 가용성 99.95%.
기술 스택은 TypeScript 기반이어야 하고, 현재 WebSocket과 Redis Pub/Sub을 고려 중이야.
```

## 검증 포인트

- [ ] **요구사항 구분**: 기능적 요구사항(채팅)과 비기능적 요구사항(10만 동시접속, 100ms 지연, 99.95% 가용성)을 구분하는가
- [ ] **여러 선택지 나열**: WebSocket 외에 SSE, gRPC 등 다른 아키텍처 패턴도 제시하는가
- [ ] **트레이드오프 분석**: 각 선택지의 장단점을 비교하는가
- [ ] **추천 + 근거**: 특정 방안을 추천하면서 왜 그것이 최적인지 근거를 제시하는가
- [ ] **비기능 요구사항 반영**: 10만 동시접속, 100ms 지연 등의 숫자가 설계에 반영되는가
- [ ] **기술 제약 존중**: TypeScript 기반이라는 제약을 준수하는가
